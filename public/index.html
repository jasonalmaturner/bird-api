Practice Bird API

This API was built with the the purpose of practicing RESTful API calls. The data is from BirdLife International. It can be downloaded from http://www.birdlife.org/datazone/userfiles/file/Species/Taxonomy/BirdLife_Checklist_Version_70.zip. See http://www.birdlife.org/datazone/info/taxonomy for more details.

The base url is http://bird-api.com.

GET

The endpoint for making a GET request is http://bird-api.com/api/birds

The following keys can be added to the url to query the bird data:

  • order
  • familyName
  • family
  • commonName
  • scientificName
  • authority
  • BirdLifeTaxonomicTreatment
  • IUCNRedListCategory2014
  • synonyms
  • alternativeCommonNames
  • SISRecID
  • SpcRecID
  • _id

Results are limited to a maximum of 50 birds. Queries are not case sensitive except for scientificName. Scientific name is both the genus and the species, with the genus being capitalized. An example of a a scientific name would be Sturnella neglecta, or the western meadowlark.


Notes about some of the keys:


BirdLifeTaxonomicTreatment

The values for BirdLifeTaxonomicTreatment are r, nr, or ur.

  • r === recognised as a species
  • nr === not recognised as a species
  • ur === under review

IUCNRedListCategory2014

The values for IUCNRedListCategory2014 are lc, nt, vu, en, cr, cr (pe), cr (pew), ew, ex, dd, nr, or ur.

  • lc === least concern
  • nt === near threatened
  • vu === vulnarable
  • en === endangered
  • cr === critically endangered
  • cr (pe) === critically endangered, possibly extinct
  • cr (pew) === critically endangered, possibly extinct in the wild
  • ew === extinct in the wild
  • ex === extinct
  • dd === data deficient
  • nr === not recognised as a species
  • ur === under review

SISRecID

SISRecID is a unique id number assigned to each taxonomic entity by IUCN/BirdLife.

SpcRecID

SpcRecID is the unique id number used in previous versions of the checklist before 2014.

http://bird-api.com/api/birds?order=passeriformes&familyName=icteridae&IUCNRedListCategory2014=cr

POST

The endpoint for making a POST request is http://bird-api.com/api/birds

The only required fields for creating and posting your bird are the order, familyName, commonName, and scientificName. Feel free to create your own unique bird. Remember that proper formatting for the scientificName is to capitalize the genus, but not the species. All scientificNames, SISRecIDs, and SpcRecIDs must be unique to successfully post. The response for successfully creating a new bird will the JSON object of the newly created bird.

The body of your POST request could look something like this:

{ "order": "jasoniformes", "familyName": "Turneridae", "family": "Turner", "commonName": "json", "scientificName": "Turner json", "authority": "me", "SISRecID": 9001, "SpcRecID": 9002 }

PUT

The endpoint for making a PUT request is http://bird-api.com/api/birds/:_id

The endpoint needs the _id of the bird you want to edit to make a PUT request. The body of the request needs to only contain the keys and corresponding values that you want to change. Any fields left out will not be changed.

The body of your PUT request could look something like this:

{ "order": "jasoniformes", "familyName": "Turneridae", "family": "Turner", "commonName": "jason", "scientificName": "Turner jason", }

DELETE

The endpoint for making a DELETE request is http://bird-api.com/api/birds/:_id

The endpoint needs the _id of the bird you want to remove from the database.

Notes:

The database is reset to the original BirdLife checklist every night. So, feel free to create and add or remove as many birds as you want!
